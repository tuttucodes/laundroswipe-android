# LaundroSwipe Scalability Strategy: Order Flow Optimization ## 1. The Challenge: Token Bottlenecks The current "Token Number" system requires manual entry by the vendor, which can slow down operations during peak hours (like a busy morning at VIT Chennai) and is prone to human error when typed. ## 2. The Proposed "Smart Flow" (Scalable) Instead of a manual token number, we move to a **Digital Handshake** model: * **Step 1: User Pre-Booking:** The student schedules a pickup in the user app. The app generates a **Dynamic QR Code** representing that specific order request. * **Step 2: Express Drop-off:** At the shop, the student simply shows the QR code. * **Step 3: Instant Admin Sync:** The vendor scans the QR code (using a mobile device or a scanner connected to the POS). This instantly: * Authenticates the user. * Loads the user's preferences (e.g., "extra starch", "eco-friendly detergent"). * Opens a **Pre-filled Bill** in the POS, ready for item count. * **Step 4: Smart Itemization:** The vendor taps the visual items (the "quick tap" grid we're building) to finalize the bill. * **Step 5: Automated Notification:** The moment the bill is "Completed," the user receives a push notification and the tracking screen updates in real-time. ## 3. Benefits of Scanned QR vs. Manual Token | Feature | Token System (Manual) | QR Handshake (Automated) | | :--- | :--- | :--- | | **Speed** | Slow (manual typing) | Instant (scan) | | **Accuracy** | High risk of typos | 100% accurate | | **Scale** | Hard to manage 100+ orders/day | Handles thousands easily | | **UX** | Feels like a utility | Feels like a modern service | ## 4. POS UI Enhancements To support this, the Admin POS will now feature: * **Quick Scan Button:** A primary action to immediately trigger the camera/scanner. * **Visual Grid Selection:** Large, identifiable icons for common items (Pant, Shirt, Saree, etc.) to minimize cognitive load for the vendor staff. * **Persistent Cart:** A side/bottom sheet that tracks the running total as items are tapped.
